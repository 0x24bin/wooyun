披露状态：

简要描述：

改一下别人的用户资料恶搞一下估计行。。。

详细说明：

“用户资料更改”页面中仅做了一个不能被称为验证的验证，导致不怀好意的人士可以通过各种修改绕过这个验证，从而达到修改任意用户资料的目的（前提是需要知道用户名）

版本：安腾网络TBMS For Oracle 3.1.9 2008-08-25

漏洞证明：

以修改电话为例

首先我们先看看尾数为40的资料，可得知电话为123456





然后我们以尾数为39的账号登录，并访问“用户更改资料”页面





多手多脚一下，用chrome的“开发人员工具”玩一玩，把39改成40

代码段：

<input type="text" name="user_id" size="20" class="text" maxlength="32" readonly="" value="xxxx217139">

<input type="hidden" name="checkeds" value="xxxx217139">

<input type="hidden" name="new_user_id" value="xxxx217139">



这样改：



然后提交。。编辑成功





回到40这个账户看看





我什么都不知道

修复方案：

换session验证咯。。。
