详细说明：

写个bat和一个reg

通过bat文件将reg文件导入

导致劫持

用户下次打开某个软件（例如QQ）的时候被替换 顺利执行恶意软件

过了360主动防御

漏洞证明：

修复方案：

给点rank

危害等级：无影响厂商忽略

厂商回复：

感谢白帽子的报告，在开启了主动防御的注册表防护后，这种方法会被拦截，不会绕过防御，因此不是360主动防御的防御绕过问题。

从截图上看，白帽子测试的似乎是Windows Server 2003服务器操作系统，目前主动防御不支持此版本的操作系统，您可以测试客户操作系统如XP , Win7等等，可以在截图上看到界面上只有360网盾相关的防护选项，在支持的操作系统版本上，是有主动防御相关（如注册表防护）的防护选项的。

目前360杀毒和360安全卫士的主动防御明确支持的主要是Windows客户操作系统，在首页上有提到是：Windows XP/ Windows Vista/Windows 7/Windows 8和Windows 8.1，对于服务器操作系统包括Windows Server 2000/2008 /2008 R2/2012/2012 R2是大部分防御功能支持，但不提供保证，对Windows Server 2003是大部分防御功能都不支持的，也没有对应的防护选项。
